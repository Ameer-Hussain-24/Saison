# 需求文档

## 简介

本功能旨在为应用的三个核心页面（任务、日程、事件）添加统一的快速跳转功能。目前任务页面已经实现了通过左上角的项目类型选择器进行页面跳转，本需求将这一功能扩展到日程和事件页面，使用户能够在这三个页面之间无缝切换。

## 术语表

- **System**: Saison 任务管理应用
- **ItemTypeSelectorBottomSheet**: 项目类型选择器底部弹窗组件
- **TaskListScreen**: 任务列表页面
- **EventScreen**: 事件列表页面
- **RoutineScreen**: 日程打卡页面
- **TopAppBar**: 页面顶部应用栏
- **Navigation**: 应用内页面导航系统

## 需求

### 需求 1

**用户故事：** 作为用户，我希望在事件页面能够快速跳转到任务或日程页面，以便我可以方便地在不同类型的项目之间切换。

#### 验收标准

1. WHEN 用户在事件页面点击顶部应用栏的标题区域，THE System SHALL 显示项目类型选择器底部弹窗
2. WHEN 用户在项目类型选择器中选择"任务"，THE System SHALL 导航到任务列表页面
3. WHEN 用户在项目类型选择器中选择"日程"，THE System SHALL 导航到日程打卡页面
4. WHEN 用户在项目类型选择器中选择"事件"，THE System SHALL 关闭底部弹窗并保持在当前页面

### 需求 2

**用户故事：** 作为用户，我希望在日程页面能够快速跳转到任务或事件页面，以便我可以方便地在不同类型的项目之间切换。

#### 验收标准

1. WHEN 用户在日程页面点击顶部应用栏的标题区域，THE System SHALL 显示项目类型选择器底部弹窗
2. WHEN 用户在项目类型选择器中选择"任务"，THE System SHALL 导航到任务列表页面
3. WHEN 用户在项目类型选择器中选择"事件"，THE System SHALL 导航到事件列表页面
4. WHEN 用户在项目类型选择器中选择"日程"，THE System SHALL 关闭底部弹窗并保持在当前页面

### 需求 3

**用户故事：** 作为用户，我希望项目类型选择器能够正确显示当前所在的页面类型，以便我知道自己当前在哪个页面。

#### 验收标准

1. WHEN 用户在任务页面打开项目类型选择器，THE System SHALL 在"任务"选项上显示选中状态指示器
2. WHEN 用户在事件页面打开项目类型选择器，THE System SHALL 在"事件"选项上显示选中状态指示器
3. WHEN 用户在日程页面打开项目类型选择器，THE System SHALL 在"日程"选项上显示选中状态指示器

### 需求 4

**用户故事：** 作为用户，我希望三个页面的顶部应用栏标题区域具有一致的交互体验，以便我能够快速识别和使用跳转功能。

#### 验收标准

1. THE System SHALL 在任务、事件和日程页面的顶部应用栏标题旁显示下拉箭头图标
2. WHEN 用户点击标题区域，THE System SHALL 提供视觉反馈（如涟漪效果）
3. THE System SHALL 在三个页面使用相同的视觉样式和交互方式
